

# Sprint 1 – Authentication & Organization Setup

**Sprint Name:** Identity & Organization Foundation

**Duration:** 1–2 weeks

**Sprint Goal:**
Enable organizations to securely create an account, authenticate users, establish an organization workspace, and manage members with role-based access.

---

# Sprint Objectives

By the end of Sprint 1, users should be able to:

* Visit a professional landing page
* Register a new organization
* Create the first Organization Owner account
* Verify their email (using Supabase Auth)
* Sign in and sign out
* Access a protected dashboard
* Create departments
* Invite team members
* Assign organizational roles
* Access only the resources they're authorized to use

# Epic 7 — Role-Based Access Control (RBAC)

Roles

| Role             | Permissions                          |
| ---------------- | ------------------------------------ |
| Owner            | Full access                          |
| Admin            | Manage organization and users        |
| Security Officer | Manage assessments and risks         |
| Manager          | Manage departments and view reports  |
| Employee         | View assigned tasks and limited data |

Implement authorization using:

* Supabase Authentication for identity
* Organization membership tables for roles
* Row Level Security (RLS) to isolate organizations
* Application-level permission checks where needed

# Definition of Done

Sprint 1 is complete when:

* A new organization can sign up.
* The first user becomes the Organization Owner automatically.
* Email verification works (or is mocked if disabled during development).
* Users can log in and log out.
* Protected routes redirect unauthenticated users.
* Organizations can create and view departments.
* Organization Owners can invite members.
* Roles are assigned and enforced.
* RLS prevents access to another organization's data.
* A basic dashboard is available after login.
* Authentication and onboarding flows are responsive, accessible, and tested.
* The application is deployed to a preview environment (e.g., Vercel) for review.

## Recommended implementation order

To minimize rework, build Sprint 1 in this sequence:

1. Configure Supabase Authentication and database tables.
2. Implement registration, login, logout, and protected routes.
3. Create the organization onboarding flow.
4. Build the dashboard layout and navigation.
5. Add department management.
6. Implement invitations and role assignment.
7. Configure and test RLS policies.
8. Polish the landing page and onboarding UX.
9. Test end-to-end and deploy a preview.

Following this order ensures each feature builds on a stable foundation and leaves you with a usable first version of CyberGuard by the end of the sprint.
