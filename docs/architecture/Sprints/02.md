
# Sprint 2 — Assessment Engine

**Sprint Goal**

Build CyberGuard's core compliance assessment engine that enables organizations to create assessment frameworks, conduct cybersecurity assessments, save drafts, resume work, and submit completed assessments for future risk analysis.

---

# Business Objectives

The Assessment Engine is the core value proposition of CyberGuard.

By the end of Sprint 2, organizations should be able to:

* Create reusable assessment templates
* Organize questions into security domains
* Assign assessments
* Complete assessments
* Save progress automatically
* Resume unfinished work
* Submit completed assessments
* Generate audit records for assessment activities

---

# Success Metrics

* Organizations can build assessment templates.
* Security domains are reusable.
* Assessment sessions persist across browser refreshes.
* Draft responses never lose user progress.
* Progress is accurately tracked.
* Assessment submission is reliable and auditable.
* RBAC and RLS protect assessment data.
* Architecture supports future scoring and reporting without redesign.

# Sprint Deliverables

At the end of Sprint 2, CyberGuard should provide:

* A configurable assessment framework.
* Reusable security domains and templates.
* Dynamic question management.
* Multi-step assessment wizard.
* Persistent draft saving.
* Resume capability.
* Real-time progress tracking.
* Review and submission workflow.
* Assessment dashboard.
* Audit logging.
* RBAC and RLS protection.

## Recommended implementation order

To minimize rework and keep the product functional after each milestone, implement the epics in this sequence:

1. **Epic 8 – Security Domains** (foundation for organizing questions)
2. **Epic 9 – Assessment Templates** (container for assessments)
3. **Epic 10 – Question Builder** (populate templates with questions)
4. **Epic 11 – Assessment Sessions** (instantiate assessments)
5. **Epic 12 – Assessment Wizard** (deliver the user-facing assessment experience)
6. **Epic 13 – Draft Management** (add persistence and resume)
7. **Epic 14 – Progress Tracking** (enhance usability and visibility)
8. **Epic 15 – Review & Submission** (complete the assessment lifecycle)
9. **Epic 16 – Assessment Dashboard** (operational overview)
10. **Epic 17 – Audit Integration** (traceability and compliance)

This structure gives you a clean, incremental roadmap where each completed epic adds tangible functionality while laying a solid foundation for **Sprint 3 (Risk Scoring & Recommendations)** without requiring architectural changes.
